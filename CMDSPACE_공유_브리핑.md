# CMDSPACE 공유 브리핑 — 17회차 박준 파트

> 운영진(요한·태극)을 위한 한 장 브리핑. **"박준이 17회차에서 어느 부분을 어떻게 맡아 실습을 이끄는가"** 를 운영 관점에서 정리.

---

## 1. 한 줄 정의

> 박준 파트는 **"공식 API가 없어도 원하는 정보를 AI가 알아서 가져오게 만드는 방법론"** 을, 수강생이 직접 손으로 돌려보는 **핸즈온**으로 진행한다. 도구 나열이 아니라 **"이럴 땐 이걸 써라"** 는 의사결정 프레임("4단 사다리")과, 그것을 Claude Code 환경으로 굳히는 **하네스화**, 마지막으로 수집물을 검증된 지식으로 쌓는 **LLM Wiki 캡스톤**까지를 하나의 흐름으로 엮는다.

---

## 2. 파트 개요 — 무엇을 다루나

박준 파트의 척추는 **정보 수집 4단 사다리**다. "위에서부터 내려와라"가 핵심 메시지.

```
1순위  공식 API 있나?        ──▶  arXiv · 법제처 · 공개데이터   (안정·합법·구조화 JSON)
2순위  검색 API로 우회        ──▶  Tavily · Exa · Brave · SerpAPI
3순위  크롤링 SaaS에 위임     ──▶  Firecrawl · Apify (안티봇·렌더링 위임)
4순위  브라우저 자동화 직접    ──▶  Playwright · cmux browser (사람 화면 그대로)
       ⭐ 위 1~4를 LLM이 대화 중 스스로 호출 → MCP/하네스로 패키징
```

여기에 더해:
- **MCP/하네스화** — 수집 도구를 LLM이 대화 중 스스로 호출하도록 패키징(`claude mcp add`, CLAUDE.md·슬래시커맨드·서브에이전트·hooks).
- **검증 루프** — 1차 출처·다중 에이전트 교차검증·인용 검증. 외부 콘텐츠는 "명령"이 아닌 "데이터"로 취급(프롬프트 인젝션 방지).
- **캡스톤(LLM Wiki)** — 위 전부를 엮어 "스스로 자라는 나만의 검증된 LLM Wiki"를 구성.

---

## 3. 진행 방식 — 어떻게 이끄나

- **바로 투입용 마스터 자료 1개** (`강의자료_MASTER.md`)로 파트 전체를 한 호흡에 진행 — 외부 수치·가격은 공식 문서 직접 검증값.
- **수강생 실습은 zip 1개로 시작** — 압축만 풀면 하네스가 이미 세팅된 상태라, 수강생은 `claude` 실행 후 `/start`만 입력하면 됨. 막히면 다시 `/start`가 현재 위치를 파악해 다음 단계를 안내(진행 속도 편차를 슬래시커맨드가 흡수).
- **무료 티어 위주** — 실습용 API 키는 전부 무료 티어로 완주 가능하게 가이드(`api_keys/00_api_key_guide.md`).
- **오프닝은 "나는 이렇게 수집한다"** — 박준 실사용 도구 지도(cmux-browser·x-clipper·web-to-markdown·korean-law MCP·graphify·notebooklm)를 4단 사다리에 매핑해 추상 개념을 실물로 연결.

## 4. 실습키트 구성 (운영진이 알아둘 것)

`실습키트/cmds17-research-lab/` (= 배포 zip)의 골격:

- `00_START_HERE.md` — 수강생 첫 화면(준비물 3개·3스텝).
- `.claude/` — 하네스: 슬래시커맨드(`/start`·`/step`·`/setup-check`·`/research-ladder`·`/wiki-build`)·권한·research-ladder 스킬.
- `code/` — 실행 가능한 단독 스크립트(arXiv·검색API·Firecrawl·Apify·Gemini·Playwright) + `mcp_add_commands.sh`.
- `guides/01~08` — 4단 사다리 단계별 가이드 + Agent SDK + Claude 하네스 + **캡스톤(LLM Wiki)**.
- `LLMWiki/` — 캡스톤 실습용 볼트. **`cmds-llm-wiki` v1.3.0 기반**(3계층 Inbox→Raw Sources 원문보존→Wiki + 검증 훅 + 17종 Web Clipper). `cd LLMWiki && claude`로 진행.

> ⚠️ 민감정보 처리: 법제처 OC키 등 개인 식별 키는 전부 `<발급받은-OC키>` placeholder로 치환 완료. 운영진/수강생은 각자 발급값으로 교체.

## 5. 역할 분담 (미팅 합의 재확인)

| 담당 | 맡은 축 | 17회차 결합 지점 |
|------|---------|------------------|
| **구요한** | 딥리서치(웹 vs API), **LLM Wiki** 연결, 하네스별 리서치 경험 차이 | 박준 캡스톤(LLM Wiki)과 직접 맞물림 — 요한 딥리서치 결과물이 박준 Wiki 파이프라인의 입력이 됨 |
| **태극** | 주력 **웹 서비스 검색** 리뷰, 브라우저 컨트롤(DevTools 녹화), 수집 가이드라인 축 | 박준 4단 사다리의 ②검색·④브라우저 단과 맞물림 — 태극의 서비스 비교가 "언제 어느 도구"의 실사용 근거 |
| **박준** | **수집 방법론(4단 사다리)·하네스·캡스톤** 실습 진행 | 세 파트를 하나의 의사결정 프레임 + 핸즈온으로 묶는 척추 역할 |

세 사람의 파트가 **딥리서치(요한) → 서비스 선택(태극) → 수집·하네스·지식화(박준)** 로 자연스럽게 이어지도록 구성.

---

## 6. 기대효과

- **수강생**: "스크래핑부터 떠올리던" 습관을 "API 먼저"로 교정 + Claude Code 하네스/MCP로 반복 작업을 환경에 굳히는 법 체득 + 검증으로 환각을 잡는 루프 습관화.
- **CMDSPACE 운영**: 가이드·모듈화되어 있어 **기업 교육 '자료조사 단계' 요구에 바로 재활용** 가능(서버 상주 + 키 발급 형태). 풀자동화는 지양하고 사람 개입 지점을 명시해 신뢰성 확보.
- **시리즈 연속성**: 이번 "정보 인입(인박스)" 편이 다음 "맥락·명령 주입" 편의 토대가 됨.
